- cкачать проект
- написать свой Dockerfile
- собрать образ, присвоить ему тег
- запустить приложение в контейнере

## Процес подготовки инфроструктуры:

### 1. Обновление списка пакетов
```
sudo apt update
```
Команда обновляет локальный индекс пакетов.

### 2. Установка Docker, Git и curl
```
sudo apt install -y docker.io git curl
```
Команда устанавливает:
- `docker.io` — Docker Engine;
- `git` — систему контроля версий;
- `curl` — утилиту для проверки HTTP-ответа.

### 3. Запуск Docker
```
sudo systemctl enable docker --now
```
Команда запускает Docker и добавляет его в автозагрузку.

### 4. Проверка Docker
```
docker --version
```
Команда показывает установленную версию Docker.

## Запуск проекта 

### 1. Сбор Docker образа
```
sudo docker build -t task1-kali-flask-home:1.0 .
```
Команда собирает Docker-образ из текущей директории и присваивает ему тег `task1-kali-flask-home:1.0`. Команда запускается через `sudo`, потому что в выполнении задания доступ к Docker daemon был получен через root.

### 2. Запуск контейнера
```
sudo docker run --rm --name task1-kali-flask-home -p 80:80 task1-kali-flask-home:1.0
```
Команда запускает контейнер и публикует приложение на `localhost:80`.

Пояснение к параметрам:
- `--rm` — удалить контейнер после остановки;
- `--name task1-kali-flask-home` — имя контейнера;
- `-p 80:80` — проброс порта `80` контейнера на порт `80` хоста.

### 3. Проверка приложения 
```
curl http://localhost/

```
Должна открыться home page приложения.

### 7. Остановка контейнера

```
Ctrl + C
```

Контейнер остановится и удалится, потому что запуск выполнялся с параметром `--rm`.
