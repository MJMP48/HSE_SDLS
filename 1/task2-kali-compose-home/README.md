Задача 2* (опционально):

Через docker compose создать связку приложения из задания 1 с БД и веб-сервером. При обращении на localhost:80 должна выдаваться home page или заглушка вашего приложения.

В качестве решения от вас ожидается ссылка на репозиторий с файлами проекта и с readme.md, где написана пошаговая инструкция как запустить ваше приложение со скриншотом ожидаемого результата.

---

## Структура проекта
```
task2-kali-compose-home/
├── app/
│   ├── app.py
│   ├── requirements.txt
│   └── Dockerfile
├── nginx/
│   └── default.conf
├── compose.yaml
├── README.md
├── REPORT.md
└── screenshots/
    └── Задание 2.png
```

### Назначение файлов
- `app/app.py` — исходный код веб-приложения;
- `app/Dockerfile` — Dockerfile для сборки контейнера приложения;
- `app/requirements.txt` — зависимости Python-приложения;
- `nginx/default.conf` — конфигурация Nginx для reverse proxy;
- `docker-compose.yml` — описание всех сервисов Compose;
- `README.md` — инструкция по запуску проекта;
- `screenshots/Задание 2.png` — скриншот ожидаемого результата.

---

## Требования
Для запуска проекта необходимо:
- Kali Linux;
- установленный Docker;
- установленный Docker Compose;
- установленный Git;
- установленный `curl`.

---


#### 1. Запуск всех сервисов через Docker Compose
```
sudo docker-compose up --build
```
Команда:
- читает конфигурацию из `docker-compose.yml`;
- собирает образ приложения из `app/Dockerfile`;
- скачивает образы PostgreSQL и Nginx, если их нет локально;
- создаёт внутреннюю сеть;
- запускает контейнеры `app`, `db` и `web`.

После запуска этот терминал будет занят логами контейнеров. Это нормальное поведение.

#### 2. Проверяем работу проекта 
```bash
curl http://localhost/
```
